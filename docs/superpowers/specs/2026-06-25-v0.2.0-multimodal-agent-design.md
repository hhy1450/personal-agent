# Personal Agent v0.2.0 — 多模态多策略 Agent 升级设计

> 日期：2026-06-25  
> 状态：实现中  
> 参考：Mobile-OwnClaw 多模态智能操作 Agent

---

## 升级总览

从 v0.1.0 纯文本顺序 Agent 升级为支持多模态感知、多策略编排、动态上下文管理、结构化动作映射和实时通信的全功能 Agent 系统。

| 模块 | v0.1.0 | v0.2.0 |
|------|--------|--------|
| LLM 层 | DeepSeek 纯文本 | DeepSeek(文本) + Qwen-VL-Max(视觉) 双 Provider |
| 工作流引擎 | Sequential only | Sequential + Parallel + Loop 三种编排 |
| 上下文管理 | 截前 500 字符 | Token 预算粗裁剪 + LLM 摘要精压缩 两层 |
| 动作映射 | LLM 自由文本 | JSON Schema 约束的结构化指令 |
| 通信 | Streamlit 同步 | FastAPI + WebSocket 实时推流 |

---

## 模块一：多 LLM Provider 层

### 改动
- LLMProvider 抽象基类增加 `supports_vision` 属性
- 新增 QwenVLProvider，用阿里云 DashScope OpenAI 兼容接口
- 新增 LLMFactory，根据 Agent 需求自动选择 Provider
- AgentConfig 增加 `requires_vision` 字段
- 配置层新增 DASHSCOPE_API_KEY 和 VISION_MODEL

### 关键设计决策
- Qwen-VL 和 DeepSeek 走同一个 OpenAI SDK，降低维护成本
- 图片消息使用 OpenAI Vision 标准格式 `image_url`
- Provider 选择逻辑在 LLMFactory 中，对上层透明

## 模块二：动态上下文管理

### 两层压缩机制
1. **第一层 Token 预算裁剪**：tiktoken 估算 token 数，超预算时保留用户意图 + 最近 N 步，从最旧步骤开始裁剪
2. **第二层 LLM 摘要压缩**：裁剪后仍超预算时，用轻量 LLM 对旧历史做摘要，替换原始内容

### 责任链模式
ContextPipeline 串联多个裁剪策略，每个策略判断达标后短路返回。

### 关键设计决策
- 摘要用 deepseek-chat（低成本），temperature=0
- 默认预算 8000 tokens，保留最近 5 步
- 独立模块，不侵入 Graph 和 Agent

## 模块三：Parallel + Loop 编排

### Planner 输出扩展
从纯列表变为带 strategy 的结构化计划，支持 group（并行组）和 loop 标记。

### 新增节点
- **FanOutNode**：将同 group 的子任务分发到并行执行边
- **FanInNode**：等待并行任务全部完成，合并结果
- **StrategyRouter**：根据 plan 的 strategy 字段选择执行路径

### Loop 实现
复用现有 QualityGate → retry 机制，但语义从"重试改正"改为"循环搜集"，增加 max_iterations 和 stop_condition 控制。

### 关键设计决策
- Executor 节点完全不改，只改拓扑
- QualityGate 不动，Router 增强
- Loop 和 Retry 共享底层循环边，通过 state 字段区分

## 模块四：结构化动作映射

### 动作定义
ActionType 枚举（search/read_file/write_file/analyze_image/summarize/generate）+
Action Pydantic 模型，JSON Schema 约束。

### ActionMapper
在 Executor 输出后插入 ActionMapper，调用轻量 LLM 将文本映射为结构化 Action。
解析成功后传给 Reviewer 做结构化校验。

### Reviewer 升级
有了结构化 Action，可以检查：
- Action 类型合理性
- 参数完整性
- Action 链闭合性

## 模块五：FastAPI + WebSocket 实时通信

### 架构
- FastAPI 作为后端服务（:8000），提供 REST API + WebSocket
- Streamlit 降级为管理面板（:8501）
- 事件总线（EventBus）连接 Graph 节点和 WebSocket handler

### WebSocket 消息协议
定义 start/step_start/tool_call/tool_result/step_done/step_review/workflow_done/error 等事件类型。

### 关键设计决策
- 事件发射非侵入式（各节点 emit，不改变核心逻辑）
- EventBus 单例，订阅者模式
- CLI 和 Web 共享同一套 API
